# codex.md

## 项目名称
Paraformer Desktop GUI

## 项目目标
基于 Qt（优先 PySide6）开发一个 Windows 桌面 exe 程序，用于本地音频转文字。  
我已经有训练好的 Paraformer 模型。  
Codex 需要按“先可运行、再可用、再产品化”的节奏逐步开发，不要一开始堆太多功能。

---

## 一、总开发原则

1. **优先技术路线**
   - 使用 `Python + PySide6`
   - 推理层复用现有 Paraformer Python 推理逻辑
   - 后续使用 `PyInstaller` 打包成 Windows exe

2. **开发原则**
   - 先做 MVP，再逐步扩展
   - 每一步都必须可运行、可验证
   - 不要把所有逻辑都堆到 UI 文件里
   - UI、控制层、推理层、导出层必须分离
   - 推理任务必须放到后台线程，不能阻塞主线程
   - 所有输出结果必须使用统一的数据结构
   - 每完成一步后，先说明改动，再给代码

3. **Codex 输出要求**
   - 每一步先给：
     - 本步目标
     - 修改/新增文件列表
     - 核心设计说明
   - 然后再给完整代码
   - 每步结束后必须给：
     - 运行方式
     - 验收方式
     - 下一步建议
   - 除非我明确要求，否则不要擅自引入大型额外框架

---

## 二、项目阶段规划

Codex 必须严格按以下阶段推进，不要跳阶段。

### Phase 1：命令行版识别核心
目标：
- 不做 GUI
- 先做命令行版本，验证 Paraformer 推理链路可用
- 支持单文件识别
- 支持导出 txt / srt / json

验收标准：
- 可以通过命令行指定音频路径
- 可以得到结构化识别结果
- 可以导出三种格式
- 代码结构清晰，后续可被 GUI 复用

### Phase 2：最小 GUI
目标：
- 用 PySide6 搭建最小可用 GUI
- 支持选择音频文件
- 支持点击“开始识别”
- 在界面中显示文本结果
- 不要求复杂配置面板

验收标准：
- GUI 能启动
- 能选择音频
- 能完成识别
- 结果能显示在界面中

### Phase 3：线程化与结果表格
目标：
- 识别任务移入后台线程
- UI 不冻结
- 显示日志、进度、状态
- 结果以表格形式展示 segment

验收标准：
- 点击识别后界面可操作
- 可看到状态变化
- 表格展示 start/end/speaker/text

### Phase 4：导出与配置
目标：
- GUI 中支持导出 txt/srt/json
- 支持基础参数配置
- 支持保存简单配置

验收标准：
- 导出按钮可用
- 导出文件格式正确
- 常用配置可修改

### Phase 5：产品化增强
目标：
- 批量识别
- 说话人区分（如果模型链路已支持）
- 热词（如果已有能力）
- 历史记录
- 打包说明

验收标准：
- 功能模块清晰
- 对已有代码侵入小
- 可逐项启用/关闭

## 五、阶段2启动性能与模型加载体验优化

### 背景问题
当前程序存在以下问题：
1. exe 启动速度慢
2. 点击加载模型后界面卡住
3. 模型加载过程没有进度提示
4. 用户无法判断程序是否仍在运行

### 本阶段目标
1. 优化程序启动流程，避免启动时执行过多重任务
2. 对重型依赖采用延迟导入
3. 模型加载必须放到后台线程执行，不能阻塞主线程
4. 增加模型加载状态管理
5. 增加进度反馈机制
6. 支持模型实例缓存，避免重复加载

### 具体要求

#### 启动优化
- `app.py` 和 `main_window.py` 启动时不得直接加载模型
- 启动时避免导入不必要的重型推理依赖
- 非关键初始化任务应延后执行
- 如使用 PyInstaller，开发阶段优先考虑 `onedir` 以改善启动速度

#### 模型加载优化
- 模型加载必须通过独立 worker 执行
- UI 线程只负责显示状态
- 点击“加载模型”后，按钮应禁用
- 模型加载期间应显示以下至少一种反馈：
  - 不确定进度条
  - 阶段性状态文本
  - 日志输出
- 模型加载完成后按钮状态恢复，界面更新为“已就绪”

#### 模型管理
建议新增 `engines/model_manager.py`，负责：
- 检查模型路径
- 加载模型
- 缓存模型实例
- 查询当前模型状态
- 卸载模型（可选）

#### 状态枚举
应定义模型状态，例如：
- NOT_LOADED
- LOADING
- READY
- FAILED

### 验收标准
1. 主窗口显示时间明显快于之前
2. 点击“加载模型”后界面仍可响应
3. 用户可以明确看到当前加载状态
4. 模型加载完成后可进入识别
5. 相同模型不重复加载

### 本阶段输出要求
Codex 本轮只完成：
1. 启动流程轻量化
2. 模型加载 worker
3. 模型状态管理
4. 基础进度反馈 UI
5. 必要的代码重构

本轮不要继续加入批量识别、热词、历史记录等功能。
